# Step 03a: Generate Primary Persona Document

**Goal:** Create detailed profile for the PRIMARY target user

---

## Purpose

Create the PRIMARY persona document - the most important user who will transform into a champion.

**Focus:** Most detail, full transformation journey, champion creation

---

## Input Required

From trigger map YAML:
- `targetGroups.primary` section
- Name, role, type, roleInFlywheel
- 6 drivingForces (3 wants, 3 fears)

---

## Template Reference

Use the comprehensive persona document template:
`templates/persona-document-template.md`

This template provides the complete structure for sections 1-13.

---

## File Naming

```
02-[Name]-the-[Role].md
```

**Example:** `02-Sarah-the-Student.md`

---

## Key Requirements

**Length:** ~250-375 lines

**Tone:**
- Rich, nuanced, human
- Not "converting" but "creating awesome"
- Natural language, storytelling

**Driving Forces:**
- Each must have **[Product] Promise**
- Show how product addresses each driver
- Be specific and actionable

**Transformation:**
- PRIMARY persona gets full BEFORE/AFTER section
- Show emotional journey, not just functional
- Use emojis to show emotional states

---

## Primary-Specific Section

**Include:** "Role in Flywheel: Creating Awesome [Personas] Who Become [Champions]"

Show:
- The natural evolution from user to champion
- What they need to see on product page
- Focus on transformation and champion creation
- How they naturally advocate after transformation
