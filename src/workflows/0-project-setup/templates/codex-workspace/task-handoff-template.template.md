# Task Handoff Template

Use this structure when assigning implementation work to Codex.

## Why

- What product or user outcome should this work achieve?
- Where does it sit in the WDS flow?
- What upstream Saga or Freya decision led here?

## What

- Exact scope
- Exact deliverable
- Acceptance criteria
- Non-goals

## How

- Relevant repo and path context
- Relevant spec files
- Relevant existing implementation files
- Technical constraints
- Reuse expectations
- Reporting thread, if any

## Definition of Done

- required code changes are implemented
- build or tests pass
- spec drift is called out
- verification is performed at the right level
- reusable learnings are captured to Design Space
