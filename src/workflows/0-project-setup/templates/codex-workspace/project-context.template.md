# Project Context

## Role of This Repo

This repository is an active WDS project.

- project specs define upstream intent
- implementation code defines current execution reality
- Whiteport Design Studio is the methodology reference, not the local source of truth

## Source Hierarchy

Use this order when determining what is true:

1. Explicit user instruction in the current session
2. Current approved task handoff
3. Relevant project specification
4. Relevant shared design-system documentation
5. Current implementation
6. WDS methodology reference

If two layers conflict, do not silently choose one. Surface the conflict.

## Local Overrides

Fill in:

- implementation stack
- key folders and landmarks
- project-specific business rules
- any recurring sources of spec drift
