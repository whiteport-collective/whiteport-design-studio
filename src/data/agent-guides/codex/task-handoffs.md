# Codex - Task Handoff Structure

The best handoff to Codex is explicit and implementation-ready.

## Required Shape

### Why

- What product or user outcome should this work achieve?
- Where does it sit in the WDS flow?
- What upstream Saga or Freya decision led here?

### What

- Exact scope
- Exact deliverable
- Acceptance criteria
- Non-goals

### How

- Relevant repo and path context
- Relevant spec files
- Relevant implementation files
- Technical constraints
- Reuse expectations
- Reporting thread if one exists

### Definition of Done

- required code changes implemented
- build or tests pass
- spec drift called out
- verification performed
- reusable learnings captured to Design Space

## Rule

Do not hand Codex a whole conversation and expect compression to substitute for decisions.
Summarize the final decision state clearly.
